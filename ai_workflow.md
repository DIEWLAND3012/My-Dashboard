# AI Development Workflow & Instructions

คู่มือและขั้นตอนการทำงานสำหรับ AI (AI Instructions) ในการพัฒนาโปรเจกต์ Dashboard Portfolio ของ Vasin เพื่อให้โค้ดมีความสมบูรณ์ สะอาด และตรงตามมาตรฐานดีไซน์ 100%

## 1. Tech Stack (เทคโนโลยีที่ใช้)
* **Frontend:** Vanilla HTML5 (Semantic Tags)
* **Styling:** Vanilla CSS3 (Modern Features: CSS Variables, CSS Grid, Flexbox, @import)
* **Scripting:** Vanilla JavaScript (ES6+, DOM Manipulation, Event Listeners)
* **Architecture:** Single Page Application (SPA) แบบ Client-side Content Switching (สลับคลาส `.active` เพื่อซ่อน/แสดงเนื้อหาโดยไม่รีโหลดหน้าเว็บ)

## 2. Coding Rules & Standards (กฎการเขียนโค้ด)
* **Naming Conventions:**
    * **CSS Class Names:** ต้องใช้รูปแบบ `kebab-case` เท่านั้น (เช่น `.bento-box`, `.film-slider-controls`)
    * **JavaScript Variables & Functions:** ต้องใช้รูปแบบ `camelCase` เท่านั้น (เช่น `filmData`, `switchPage()`)
* **CSS Standards:**
    * ห้ามเขียน CSS แบบ Inline ในไฟล์ HTML ให้แยกเขียนในไฟล์ CSS ตามโครงสร้างโฟลเดอร์ที่กำหนดไว้
    * ใช้ CSS Variables สำหรับ Design Tokens (สี, ฟอนต์, ขอบมน) เพื่อให้ง่ายต่อการแก้ไขส่วนกลาง
    * ห้ามล็อกความสูงแบบตายตัว (`height`) บนกล่องข้อความ ให้ใช้ `padding` หรือ `min-height` เพื่อรองรับความยาวตัวอักษรที่ยืดหยุ่น
* **JavaScript Standards:**
    * เขียนโค้ดแบบ Modular แยกส่วนการทำงาน (Separation of Concerns) ตามโครงสร้างไฟล์อย่างเคร่งครัด
    * ห้ามเขียนโค้ดที่ตัดทอน (หลีกเลี่ยงการใช้ `// ... rest of code`) ให้ส่งโค้ดเวอร์ชันเต็มที่พร้อมใช้งานเสมอ
    * ใส่คอมเมนต์อธิบายสั้นๆ (Comments) ในฟังก์ชันที่ซับซ้อน

## 3. Useful Commands (คำสั่งสำหรับการรันโปรเจกต์)
เนื่องจากโปรเจกต์นี้เป็น Vanilla Web (ไม่มี Build Step) สามารถใช้คำสั่งเหล่านี้ในการทดสอบระบบเครื่องโลคอล (Local Environment):

* **Python (คำสั่งพื้นฐานติดเครื่อง):**
    ```bash
    python -m http.server 3000
    ```
    *(เข้าใช้งานผ่านบราวเซอร์ที่: `http://localhost:3000`)*
* **Node.js (หากติดตั้ง `serve` ไว้):**
    ```bash
    npx serve
    ```
* **VS Code Tool:** แนะนำให้ผู้ใช้งานรันผ่าน Extension **"Live Server"** เพื่อให้เกิดการ Auto-refresh หน้าเว็บทุกครั้งที่บันทึกไฟล์

## 4. Pre-Modification Workflow (ขั้นตอนที่ AI ต้องทำก่อนเริ่มเขียน/แก้ไขโค้ด)
เมื่อผู้ใช้งานสั่งให้แก้ไขหรือเพิ่มฟีเจอร์ใหม่ **AI จะต้องทำตามขั้นตอนเหล่านี้ทุกครั้ง ห้ามข้ามสเต็ปเด็ดขาด:**

### ขั้นตอนที่ 1: ตรวจสอบและอ่านคัมภีร์ดีไซน์ (`project_detail.md`)
* ก่อนจะเขียนโค้ดทุกครั้ง AI ต้องเปิดอ่านไฟล์ `project_detail.md` เพื่อทบทวนค่าสี คอนทราสต์ ขนาดความโค้งมน (`border-radius`), และลักษณะอนิเมชัน (ความเร็ว 200ms-400ms) เพื่อให้สไตล์ใหม่กลมกลืนกับระบบเดิม

### ขั้นตอนที่ 2: อ่านไฟล์โค้ดต้นทางและไฟล์ที่เกี่ยวข้อง (Read & Understand)
* หากผู้ใช้งานสั่งแก้ไขไฟล์ใดไฟล์หนึ่ง AI ต้องขออ่านหรือจำลองโครงสร้างปัจจุบันของไฟล์นั้นก่อน
* ต้องตรวจสอบไฟล์ที่เชื่อมโยงกันเสมอ เช่น:
    * ถ้าจะแก้ปุ่มสไลด์หนังใน `js/components/filmSlider.js` -> AI ต้องอ่านโครงสร้างข้อมูลใน `js/data/filmsData.js` และโครงสร้าง HTML ใน `index.html` ช่วงของ `#film-page` ก่อนทุกครั้ง

### ขั้นตอนที่ 3: วางแผนโครงสร้างการแก้ไข (Propose & Draft)
* บอกผู้ใช้งานอย่างชัดเจนว่า โค้ดที่จะเขียนใหม่หรือแก้ ต้องนำไปใส่ในโฟลเดอร์ใดและไฟล์ใดตามโครงสร้างโปรเจกต์หลัก (เช่น `css/pages/gallery.css` หรือ `js/components/navigation.js`)
* ห้ามสร้างไฟล์ใหม่ที่อยู่นอกเหนือโครงสร้างที่ตกลงกันไว้ เว้นแต่จะได้รับความยินยอมจากผู้ใช้งาน

### ขั้นตอนที่ 4: เขียนโค้ดตัวเต็มและอธิบายวิธีการใช้งาน (Deliver Clean Code)
* ส่งโค้ดที่เขียนเสร็จสมบูรณ์ ไม่มีบรรทัดที่ละไว้ในฐานที่เข้าใจ
* อธิบายสั้นๆ ว่าโค้ดใหม่นี้จะเข้าไปโต้ตอบ (Interact) กับ UI State อย่างไร (เช่น Hover State, Active State ตามที่ระบุไว้ในรายละเอียดดีไซน์)
