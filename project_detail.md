## 1. Design & Layout (การออกแบบและการจัดวาง)
* [cite_start]**ขนาดหน้าจอเริ่มต้น (Base Resolution):** ออกแบบตาม Wireframes ใน Figma ที่ขนาด 1440 x 1024 พิกเซล [cite: 1]
* [cite_start]**การรองรับหน้าจอ (Responsive):** เว็บไซต์ต้องมีความยืดหยุ่น แสดงผลได้ดีในทุกอุปกรณ์ โดยยังคงความสวยงามและใช้งานง่าย [cite: 1]
  * [cite_start]*Mobile/Tablet:* แถบ Sidebar ด้านซ้ายจะหดตัวเก็บเหลือเพียงไอคอน (ซ่อนข้อความ) หรือปรับเปลี่ยนเป็นเมนูด้านล่าง (Bottom Navigation) แทน เพื่อเพิ่มพื้นที่แสดงผลฝั่งขวาให้เต็มตาที่สุด [cite: 16]
* **Design System & Styling:**
  * [cite_start]**ความโค้งมน (Border Radius):** กรอบนอก (Outer) = `30px`, กรอบใน (Inner) = `10px` [cite: 16, 17]
  * [cite_start]**ฟอนต์ (Typography):** ใช้ `LINE Seed JP` สำหรับภาษาอังกฤษ [cite: 17]
  * **ชุดสี (Color Palette):**
    * [cite_start]สีพื้นหลังหลัก (Background): `#455C6B` [cite: 17]
    * [cite_start]สีพื้นบอร์ด (Board): ไล่สี Linear Gradient จาก `#50798A` ไป `#ECCDBC` (จากซ้ายไปขวา) [cite: 17]
    * [cite_start]สีขาว (White Base / Elements): `#FFFCFF` [cite: 17]
    * [cite_start]สีดำและตัวหนังสือ (Black / Text): `#000000` [cite: 17]

## 2. Animation & Transitions (การเคลื่อนไหวและเอฟเฟกต์)
* [cite_start]**ความเร็ว (Speed & Smoothness):** ใช้ระยะเวลา 200ms ถึง 400ms ควบคู่กับ Timing function แบบ `ease-out` หรือ `cubic-bezier` เพื่อให้สมูท เป็นธรรมชาติ และไม่รู้สึกว่าเว็บอืด [cite: 1, 2]
* [cite_start]**สไตล์การเคลื่อนไหว:** เน้น "น้อยแต่มาก" (Subtle) หลีกเลี่ยงเอฟเฟกต์รุนแรง แต่ใช้การขยับเล็กน้อยที่มีความหมาย (Functional motion) [cite: 2]
* [cite_start]**การสลับหน้าหลัก (SPA Transition):** มีแถบ Smooth Progress Bar ปรากฏขึ้นที่ด้านบนของเว็บระหว่างการเปลี่ยนหน้า เพื่อบอกสถานะการโหลดข้อมูล [cite: 12]
* **การเปลี่ยนเนื้อหา (Main Content Transition):**
  * [cite_start]*เปลี่ยนหน้าทั่วไป (เช่น Dashboard ไป Gallery):* ใช้ Cross-Fade โดยเนื้อหาเก่าจะค่อยๆ Fade-out และขยับเล็กลงเล็กน้อย พร้อมกับเนื้อหาใหม่ที่ Fade-in และขยายขึ้นเล็กน้อย [cite: 5]
  * [cite_start]*เปลี่ยนไปหน้ารายละเอียด (เช่น Profile ไป VIEW FULL PROFILE):* แผงโปรไฟล์หลักที่เชื่อมต่อกันจะขยายตัวออกอย่างนุ่มนวล ให้ความรู้สึกว่าขยายออกมาจากส่วนเดิม [cite: 4]
* [cite_start]**โหลดแบบขั้นบันได (Staggered Loading):** ลิสต์หรือการ์ดต่างๆ (เช่น ปี 2026, 2025 ในหน้า Gallery) ห้ามโผล่มาพร้อมกันหมด แต่จะค่อยๆ Fade-in โผล่ขึ้นมาทีละนิดโดยเว้นระยะ (Delay) 50ms ต่อรายการ [cite: 6, 12]
* [cite_start]**Title Animation:** ข้อความหัวเรื่อง (เช่น 'Gallery 2023-NOW') จะปรากฏแบบสั้นๆ ด้วย Typewriter effect หรือสไลด์ขึ้น [cite: 13]

## 3. UI States & Interaction (สถานะการโต้ตอบ)
* [cite_start]**Sidebar Navigation (แถบเมนูคงที่):** [cite: 3, 14]
  * [cite_start]*Default State:* ไอคอนและข้อความโปร่งใสเล็กน้อย (`opacity: 0.6`) [cite: 14]
  * [cite_start]*Hover State:* ไอคอนขยายขึ้นเล็กน้อย, ตัวอักษรหนาขึ้นเบาๆ, `opacity` เปลี่ยนเป็น 1.0 และมีวงกลมสีขาวจางๆ (`rgba(255,255,255,0.15)`) ปรากฏขึ้น แถบไฮไลท์สไลด์ตามเมาส์อย่างนุ่มนวล [cite: 7, 14]
  * [cite_start]*Active State:* วงกลมเป็นสีขาวทึบ, ไอคอนเปลี่ยนเป็นสีเข้ม (สีพื้นหลัง) เพื่อเพิ่ม Contrast [cite: 14] [cite_start]หากมีการเปลี่ยนหน้า วงกลมขาวจะใช้แอนิเมชัน "Slide & Stretch" สไลด์เลื่อนแล้วยืดออกไปหาเมนูใหม่ [cite: 14]
* **ปุ่มหลัก (Primary Buttons):** (เช่น VIEW FULL PROFILE, WATCH NOW!)
  * [cite_start]*Hover State:* สีพื้นหลังเปลี่ยนอย่างนุ่มนวล, เพิ่มความเข้มของเงา (`box-shadow`) เพื่อสื่อว่ากดได้ [cite: 8, 9]
* [cite_start]**ปุ่มลูกศร (หน้า Film):** ระบบเป็น Infinite Loop วนลูปไม่รู้จบ จึงไม่มีสถานะ Disabled [cite: 14]
  * [cite_start]*Hover State:* ปุ่มขยาย `scale(1.05)` และลูกศรด้านในขยับเดินหน้า 2-3px ตามทิศทาง หรือเปลี่ยนสี [cite: 11, 14]
  * [cite_start]*Pressed State:* ปุ่มหดตัว `scale(0.95)` เพื่อสร้าง Tactile Feedback เหมือนการกดปุ่มจริง [cite: 14, 15]
  * [cite_start]*Looping State:* เมื่อสไลด์จนวนกลับหน้าแรก เนื้อหาทั้งหมดจะสไลด์เลื่อนกลับไปเริ่มต้นทางซ้ายอย่างรวดเร็ว (Fast Rewind) แบบลื่นไหล [cite: 15]
* [cite_start]**ปุ่มย้อนกลับ (Back Button):** * *Hover State:* ตัวลูกศรและปุ่มขยับถอยหลังไปทางซ้ายเล็กน้อย สีเปลี่ยนไป ไอคอนขยาย [cite: 9, 15, 16]
  * [cite_start]*Focus State:* หากใช้คีย์บอร์ด (Tab) จะมีเส้น Outline ล้อมรอบชัดเจน (Accessibility) [cite: 16]
* **การ์ดเนื้อหา (Content Cards):** (เช่น Random content, Image 1:1)
  * [cite_start]*Hover State:* ยกตัวขึ้นเล็กน้อย (`TranslateY -4px`) และมีเงานุ่มนวลขึ้น หรือรูปภาพภายในค่อยๆ ขยายตัว (scale-in) แบบประณีต [cite: 10, 11]
* **ระบบ Gallery Pop-up (เมื่อคลิก Image 1:1):**
  * [cite_start]*Display:* การ์ดย่อจะครอปรูปให้เท่ากัน แต่พอกดแล้วรูปจะขยาย Pop-up ตรงกลางจอตามอัตราส่วนจริง (4:3, 3:4, 2:3, 3:2) [cite: 15]
  * [cite_start]*Effect:* พื้นหลังจะมืดลง 80% (Backdrop Blur) [cite: 15]
  * [cite_start]*Cinematic Detail:* แสดงข้อมูลเชิงลึกแบบมืออาชีพด้านข้างภาพ เช่น รุ่นกล้อง (Sony a7C II), เลนส์, ค่าสปีดชัตเตอร์ [cite: 15]
* [cite_start]**สถานะรอโหลด (Loading State):** ไม่ใช้ไอคอนหมุนๆ แต่ใช้ "Shimmer Effect" แสดงกล่องโครงร่างสีเทาอ่อนและมีไฮไลท์แสงสีขาววิ่งผ่านจากซ้ายไปขวา [cite: 15]

## 4. Core Systems (ระบบหลัก)
* [cite_start]**SPA (Single Page Application):** ใช้ JavaScript ซ่อน/แสดงหน้าต่างๆ `<section>` ผ่านคลาส `active` โดยจะไม่มีการโหลดหน้าใหม่เลย [cite: 16]
* [cite_start]**ภาษา (Localization):** ยังไม่พัฒนาระบบ 2 ภาษา ให้ยึดเริ่มต้นใช้งานเป็นภาษาอังกฤษ (EN) ก่อน [cite: 16]
